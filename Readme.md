* Наставник: [Андрей Грачев](https://htmlacademy.ru/profile/id252055)

Опубликованная версия — <a href="https://kalininmax.github.io/device" target="_blank">kalininmax.github.io/device</a>

## Техническое задание

### 1\. Общие технические требования

1.1. Стандарты вёрстки: HTML, CSS, прогрессивное улучшение.<br>
1.2. Сетка: определена в макете.<br>
1.3. Адаптивность вёрстки: нет.<br>
1.4. Используемые фреймворки: нет.<br>
1.5. Кроссбраузерность: Chrome, Firefox, Safari.<br>
1.6. Типографика: частично определена в макете, прочее — на усмотрение разработчика.<br>
1.7. Используемые шрифты: Gilroy, Open Sans. Шрифты есть в папке проекта и на [WebFont.ru](http://webfont.ru/).<br>
1.8. С макетом предоставлен стайлгайд, содержащий прорисовку состояний элементов интерфейса. При любых расхождениях с макетами он должен иметь наивысший приоритет.

### 2\. Обязательные требования

#### Все макеты:

3.1. Контентная область центрируется и не может быть уже макетной ширины. Фоны, которые упираются в края макета, должны тянуться на всю страницу.<br>
3.2. Авторизованному и неавторизованному посетителю показывается разный вид блока авторизации (смотрите страницы **catalog** и **index**).<br>
3.3. В блоке авторизованного посетителя имя и иконка пользователя являются ссылкой на профайл.<br>
3.4. Логотип Академии в подвале ведёт на [лендинг интенсива «HTML и CSS. Профессиональная вёрстка сайтов»](https://htmlacademy.ru/intensive/htmlcss).

#### Index:

3.5. Блоки с названиями разделов («виртуальная реальность», «моноподы для селфи», ...) являются ссылками на разделы каталога.<br>
3.6. Промо-слайдер: слайдер. Вёрстка всех слайдов обязательна.<br>
3.7. Блок «Сервисы» («доставка», «гарантии», «кредит»): слайдер. Вёрстка всех слайдов обязательна.<br>
3.8. Блок карты: достаточная реализация — обычное изображение, клик по ней приводит к переходу на сервис карт.<br>
3.9. Вёрстка модального окна с формой обратной связи обязательна.

#### Catalog:

3.10. Логотип — это ссылка на главную страницу.<br>
3.11. Фильтр: верстать с помощью формы, кнопка «Показать» отвечает за отправку формы на адрес `https://echo.htmlacademy.ru/`.<br>
3.12. Блок «Стоимость»: при взаимодействии с любым из маркеров в качестве указателя допускается использовать любой подходящий тип курсора. Делать маркеры интерактивными не обязательно, стоимость меняться не должна.<br>
3.13. При наведении на карточку товара появляются кнопки «В корзину» и «Добавить к сравнению» (смотрите страницу **styleguide**).<br> Ссылкой на отдельную страницу товара является только название товара. Отдельную страницу верстать не нужно.<br>
3.14. Количество товаров в правом блоке может быть любым, добавление товаров не должно ломать страницу.<br>
3.15. У любого товара может быть метка «new».

### 4\. Дополнительные требования

#### Index:

4.1. Промо-слайдер: оживление слайдера. Смена слайдов в слайдере должна происходить мгновенно, без промежуточных состояний и анимации.<br>
4.2. Блок «Сервисы» («доставка», «гарантии», «кредит»): оживление слайдера. Слайдер с табами работает аналогично промо-слайдеру: смена слайда происходит мгновенно при клике на соответствующий таб.<br>
4.3. Блок карты: обычное изображение. Клик по ней открывает модальное окно с полноразмерной интерактивной картой. Окно позиционируется относительно вьюпорта, а не страницы. Реализация по желанию.<br>
4.4. Клик по кнопке «Напишите нам» открывает модальное окно с формой обратной связи. Окно позиционируется относительно вьюпорта, а не страницы.

#### Catalog:

4.5. Фильтр: клик по кнопке «Показать» отправляет форму.
